# Production Brief

## Core message
Arbitrum proposals move across multiple governance stages and platforms. This product gives users one place to track that lifecycle clearly.

## Positioning
Use language like:
- "Arbitrum DAO Governance Tracker"
- "A unified index for proposal lifecycle visibility"
- "A governance hub, not a voting interface"

Avoid language like:
- "Trade"
- "Participate here"
- "Governance super app"
- "All-in-one voting platform"

## Tone
- Calm
- Trustworthy
- Technical but accessible
- Premium, minimal, data-product feel
- No overpromising

## Narrative arc
1. Governance is fragmented
2. Following one proposal is harder than it should be
3. The tracker unifies visibility
4. Users can quickly see stage, status, and source links

## Visual direction
- Neutral dark intro or soft light background
- Minimal motion graphics with clear arrows and cards
- Strong typography
- Smooth zooms and subtle transitions
- Real UI footage in second half

## Video structure
- 0–30s: animated concept and problem framing
- 30–72s: real product demo

## Visual motifs
- One proposal card traveling through stages
- Forum → Snapshot → Tally pathway
- Messy multi-tab workflow collapsing into one clean row
- Status labels and source-link clusters

## Editing rules
- Never imply users can vote from this product
- Keep table views readable; do not over-scroll too fast
- Use the product name early on reveal
- Let UI breathe for 1–2 seconds on important beats

## Recommended music
- Light, modern, understated tech/electronic bed
- No aggressive drops
- Build slightly at product reveal

## Recommended VO style
- Confident
- Conversational
- Slightly analytical
- Medium pace
